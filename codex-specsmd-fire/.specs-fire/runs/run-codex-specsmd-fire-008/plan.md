# Implementation Plan: Frontend Foundation

## Work Item

`frontend-foundation` for intent `friccion-cero`.

## Approach

Create the Angular frontend under `friccion-cero/frontend` using Angular routing, PrimeNG, a simple application shell, configurable API base URL, shared TypeScript models, and API services aligned with the backend contracts already implemented.

This run will build the foundation only. The detailed friction, initiative, and dashboard screens stay for their dedicated work items, but their routes will exist with lightweight placeholder views so navigation and structure are ready.

## Files and Folders to Create

- `friccion-cero/frontend/`
  - Angular application scaffold.
- `friccion-cero/frontend/src/app/app.config.ts`
  - Application providers, router, HTTP client, animations, and PrimeNG setup.
- `friccion-cero/frontend/src/app/app.routes.ts`
  - Routes for dashboard, frictions, friction detail, friction form, initiatives, and initiative detail.
- `friccion-cero/frontend/src/app/core/api/`
  - API client services for frictions, initiatives, and dashboard summary.
- `friccion-cero/frontend/src/app/core/models/`
  - Shared frontend models matching backend response contracts.
- `friccion-cero/frontend/src/app/layout/`
  - App shell with navigation for MVP screens.
- `friccion-cero/frontend/src/app/features/*`
  - Minimal routed placeholder components for future screen work items.
- `friccion-cero/frontend/src/environments/`
  - `environment.ts` and `environment.development.ts` with configurable `apiBaseUrl`.

## Files to Modify

- Generated Angular configuration files as needed:
  - `package.json`
  - `angular.json`
  - `src/styles.scss`
- `.specs-fire/intents/friccion-cero/work-items/frontend-foundation.md`
  - Mark acceptance criteria complete after verification.

## Commands

Planned scaffold and setup:

- `npx @angular/cli new frontend --directory friccion-cero/frontend --routing --style=scss --skip-git`
- `npm install primeng primeicons @angular/animations`

Verification from `friccion-cero/frontend`:

- `npm run build`
- `npm test -- --watch=false` if the generated test setup is available and stable.
- Start the dev server and provide the local URL.

## UI Foundation

- Use a restrained operational-tool layout rather than a marketing page.
- Provide a top/side navigation shell with links for:
  - Dashboard
  - Frictions
  - New friction
  - Initiatives
- Use PrimeNG components directly for navigation, buttons, cards/panels, tables placeholders, and route content containers.
- Keep placeholder pages functional and route-focused without pretending the final screens are done.

## API Client Foundation

Create services with methods matching current backend endpoints:

- `DashboardApiService.getSummary()` -> `GET /api/dashboard`
- `FrictionApiService.list/get/create/update/delete/listComments/createComment`
- `InitiativeApiService.list/get/update/delete/createFromFriction`

The API base URL will come from Angular environment files, defaulting to `http://localhost:3000`.

## Tests

Minimum verification for this run:

- Angular build passes.
- Basic generated tests pass if available.
- Dev server starts locally.
- Routes compile and the shell renders.

## Decisions

- Keep frontend architecture simple: `core`, `layout`, and `features`.
- Use Angular standalone-style routing/providers if generated by the current CLI.
- Do not implement full data-heavy screens in this run; create the foundation they will use.
