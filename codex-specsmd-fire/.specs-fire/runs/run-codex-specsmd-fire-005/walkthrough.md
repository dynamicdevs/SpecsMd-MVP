## Key Implementation Details

### 1. Derived Fields

Create and update operations call `enrichFrictionInput`, so monthly hours, estimated cost, category, automation potential, and priority are centralized.

### 2. Validation

Invalid request bodies return `400` with a compact issue list from Zod.

### 3. Missing Records

Missing friction IDs return `404` through `HttpError`.

### 4. Comments

Basic comment creation/listing is nested under `/api/frictions/:id/comments`.

## Decisions Made

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Route shape | `/api/frictions` with nested comments | Simple REST-like MVP contract. |
| Controller style | Thin handlers plus service | Keeps request/response separate from domain coordination. |
| Update behavior | Merge existing + input then recalculate | Keeps derived fields consistent when inputs change. |
