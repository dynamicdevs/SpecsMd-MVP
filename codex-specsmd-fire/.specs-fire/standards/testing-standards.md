**Target**: 0% until a test runner is introduced

## Mock Strategy

**Approach**: Use temporary directories for filesystem mutation tests.

**Guidelines**:

- Avoid mutating the real package during automated tests.
- Keep fixture state minimal and readable.
- Verify generated YAML and Markdown with structured or snapshot checks.

## Test Data

**Strategy**: Small fixture workspaces that mirror `.specs-fire` schema.

**Guidelines**:

- Keep fixture names descriptive.
- Include at least one pending work item and one active run fixture.
- Avoid private or user-specific paths in committed fixtures.
