# Tech Stack

## Overview

Fricción Cero es una aplicación fullstack monorepo. El backend expone una API REST con
FastAPI y persiste en SQLite mediante SQLModel. El frontend es una SPA Angular con PrimeNG
que consume la API real (sin datos mock en la versión final del MVP).

> Nota: el stack de backend originalmente solicitado era .NET 8. Se cambió a **FastAPI + Python**
> por decisión del usuario. Ver `.specs-fire/standards/system-architecture.md` (Key Decisions).

## Core Technologies

### Language

**Primary**: Python (backend) + TypeScript (frontend)
**Version**: Python 3.12 / TypeScript 5.x

### Framework

**Framework**: FastAPI (backend) + Angular (frontend)
**Version**: FastAPI 0.11x / Angular 18+
**Rationale**: FastAPI da OpenAPI/Swagger automático, validación con Pydantic y arquitectura
modular simple. Angular + PrimeNG cumple el requisito original de UI (formularios reactivos,
servicios HTTP por feature, componentes por feature).

### Runtime

**Runtime**: CPython 3.12 (backend) / Node.js (build frontend) + navegador (runtime frontend)
**Version**: Python 3.12.10 / Node 24

## Data Layer

### Database

**Type**: Relacional (archivo local)
**Database**: SQLite
**Version**: SQLite 3

### ORM / Data Access

**Tool**: SQLModel (sobre SQLAlchemy + Pydantic)
**Rationale**: Un solo modelo sirve como entidad ORM y base para los schemas; menos boilerplate
para un MVP. Las migraciones del MVP se manejan por `create_all` (Alembic queda como evolución futura).

### Caching

Not configured.

## API Layer

### API Style

**Style**: REST (JSON)

### Documentation

**Tool**: Swagger UI / OpenAPI (generado automáticamente por FastAPI en `/docs`)

## Frontend

### UI Framework

**Framework**: Angular + PrimeNG
**Version**: Angular 18+

### Styling

**Approach**: PrimeNG + PrimeFlex + tema PrimeNG (CSS utilitario)

### State Management

**Solution**: Servicios Angular + RxJS (sin store global; suficiente para el MVP)

## Infrastructure

### Hosting

Not configured (posible evolución futura con Docker).
