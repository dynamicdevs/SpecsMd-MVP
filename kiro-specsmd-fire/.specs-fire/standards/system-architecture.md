# System Architecture

## Overview

Fricción Cero es una aplicación fullstack cliente-servidor para gestión de fricciones operativas. Arquitectura modular simple, diseñada para ser rápida de montar en el MVP pero extensible a futuro.

## System Context

```
┌─────────────┐         ┌──────────────────┐         ┌────────────┐
│   Usuario   │ ──HTTP──│  Angular Frontend │ ──REST──│  FastAPI   │
│  (Browser)  │         │   (PrimeNG)      │         │  (Backend) │
└─────────────┘         └──────────────────┘         └─────┬──────┘
                                                           │
                                                     ┌─────▼──────┐
                                                     │   SQLite    │
                                                     │   (Data)    │
                                                     └────────────┘
```

### Users

- **Usuario interno**: Empleado o líder de área que registra y gestiona fricciones operativas

### External Systems

- Ninguno en MVP (100% self-contained)

## Architecture Pattern

**Pattern**: Modular simple (routers / services / models / schemas)
**Rationale**: FastAPI promueve naturalmente esta separación. Sin capas innecesarias para un MVP. Directo y limpio.

### Capas

```
┌─────────────────────────────────────┐
│       Routers (API endpoints)        │  ← HTTP handling, validation via Pydantic
├─────────────────────────────────────┤
│       Services (Business Logic)      │  ← Cálculos, clasificación, priorización
├─────────────────────────────────────┤
│       Models + Database              │  ← SQLAlchemy models, SQLite
└─────────────────────────────────────┘
```

## Data Flow

### Crear Fricción

```
Usuario → Angular Form → HTTP POST /api/v1/frictions
  → Router → Pydantic schema validation (automática)
  → calculation_service.calculate_monthly_hours()
  → calculation_service.calculate_monthly_cost()
  → classification_service.determine_automation_potential()
  → priority_service.calculate_priority()
  → SQLAlchemy session.add() → SQLite
  → Return FrictionResponse (Pydantic schema)
```

## Key Components

| Component | Responsibility |
|-----------|---------------|
| frictions router | CRUD endpoints para fricciones |
| initiatives router | CRUD endpoints para iniciativas |
| dashboard router | Métricas agregadas |
| calculation service | Cálculo de horas perdidas y costo mensual |
| classification service | Clasificación de categoría y potencial de automatización |
| priority service | Priorización automática |

## Non-Functional Requirements

### Performance

- Response time API: < 100ms para operaciones CRUD (FastAPI + SQLite es muy rápido)
- Dashboard load: < 500ms

### Security (MVP)

- CORS configurado para frontend local
- Input validation automática via Pydantic
- Sin autenticación en MVP (app interna)

### Scalability

- SQLite → PostgreSQL cuando se necesite concurrencia (cambio en config, no en código)
- Monolito → servicios separados no necesario en MVP

## Constraints

- Sin APIs externas en MVP
- Sin autenticación/autorización en MVP
- SQLite como base de datos (archivo local)
- Valor hora configurable via .env / config
