# Testing Standards

## Overview

Estrategia de testing pragmática para MVP. Foco en lógica de negocio crítica (cálculos, clasificación, priorización) y flujos principales de la API.

## Testing Framework

### Backend

- **Framework**: pytest
- **HTTP Testing**: httpx (TestClient de FastAPI)
- **Fixtures**: pytest fixtures

### Frontend

- **Framework**: Jasmine + Karma (default Angular)
- **E2E**: No requerido para MVP

## Test Types

| Type | Tool | Location | When to Use |
|------|------|----------|-------------|
| Unit (Backend) | pytest | `backend/tests/` | Servicios, cálculos, reglas |
| Integration (Backend) | pytest + TestClient | `backend/tests/` | Endpoints API |
| Unit (Frontend) | Jasmine | `*.spec.ts` co-located | Services, pipes, lógica |

## Coverage Requirements

**Target MVP**: 70% en servicios de cálculo y clasificación
**Crítico (must have)**:
- calculation.py (horas perdidas, costo mensual)
- classification.py (categoría, potencial automatización)
- priority.py (priorización automática)

## Test Naming

**Pattern**: `test_function_scenario_expected`

**Examples**:
- `test_calculate_monthly_hours_valid_input_returns_correct`
- `test_classify_friction_repetitive_task_returns_high_automation`
- `test_get_priority_high_cost_and_pain_returns_high`

## Running Tests

```bash
# Backend - Run all tests
pytest

# Backend - With coverage
pytest --cov=app --cov-report=html

# Backend - Specific file
pytest tests/test_calculation.py

# Frontend - Run tests
ng test

# Frontend - Single run (CI)
ng test --no-watch --browsers=ChromeHeadless
```
