# Tech Stack

## Overview

Fricción Cero es una aplicación fullstack para gestión de fricciones operativas. Arquitectura cliente-servidor con Angular en frontend y FastAPI (Python) en backend, SQLite como base de datos inicial.

## Core Technologies

### Backend

**Language**: Python 3.11+
**Framework**: FastAPI
**Rationale**: Ultra rápido de montar, docs OpenAPI automáticas, validación con Pydantic incluida, ideal para MVP de CRUD con lógica de cálculo.

### Frontend

**Framework**: Angular 18+
**UI Library**: PrimeNG
**Language**: TypeScript 5.x
**Rationale**: Framework robusto para aplicaciones empresariales, PrimeNG provee componentes de datos (tablas, charts, forms) out-of-the-box.

## Data Layer

### Database

**Type**: Relational (SQL)
**Database**: SQLite
**Rationale**: Zero-config, archivo local, perfecto para MVP. Migración futura a PostgreSQL sin cambios en el código (via SQLAlchemy).

### ORM / Data Access

**Tool**: SQLAlchemy 2.0
**Rationale**: ORM maduro de Python, soporte SQLite/PostgreSQL transparente, type hints con mapped_column.

## API Layer

### API Style

**Style**: REST
**Format**: JSON
**Versioning**: URL path prefix (/api/v1/)

### Documentation

**Tool**: Swagger / OpenAPI (automático con FastAPI, zero-config)

## Infrastructure

### Package Manager

- **Backend**: pip + requirements.txt (o uv para velocidad)
- **Frontend**: npm

### Build Tool

- **Backend**: uvicorn (dev server), no build step necesario
- **Frontend**: Angular CLI (ng build)

### Linting

- **Backend**: Ruff (linter + formatter, reemplaza flake8/black/isort)
- **Frontend**: ESLint + Angular ESLint

### Formatting

- **Backend**: Ruff format
- **Frontend**: Prettier

## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Backend framework | FastAPI | Swagger gratis, Pydantic valida DTOs sin código extra, menos boilerplate |
| Database | SQLite | Zero-config para MVP, migrable a PostgreSQL |
| ORM | SQLAlchemy 2.0 | Maduro, type-safe con 2.0, migrations con Alembic |
| UI Library | PrimeNG | Componentes de datos ricos (Table, Chart, Dialog) |
| Architecture | Modular simple | routers / services / models / schemas — sin sobreingeniería |
| State mgmt | Angular Signals + Services | Suficiente para MVP sin NgRx overhead |
