# Coding Standards

## Overview

Estándares de código para Fricción Cero. Prioridad: legibilidad, consistencia y simplicidad para el MVP.

## Backend (Python / FastAPI)

### Naming Conventions

| Element | Convention | Example |
|---------|------------|---------|
| Modules/files | snake_case | `friction_service.py` |
| Classes | PascalCase | `FrictionService` |
| Functions | snake_case | `calculate_monthly_hours()` |
| Variables | snake_case | `friction_count` |
| Constants | UPPER_SNAKE_CASE | `DEFAULT_HOURLY_RATE` |
| Pydantic schemas | PascalCase | `FrictionCreate`, `FrictionResponse` |

### Project Structure

```
backend/
├── app/
│   ├── __init__.py
│   ├── main.py                 # FastAPI app, CORS, startup
│   ├── config.py               # Settings via pydantic-settings
│   ├── database.py             # SQLAlchemy engine + session
│   ├── models/                 # SQLAlchemy models
│   │   ├── __init__.py
│   │   ├── friction.py
│   │   └── initiative.py
│   ├── schemas/                # Pydantic DTOs (request/response)
│   │   ├── __init__.py
│   │   ├── friction.py
│   │   └── initiative.py
│   ├── routers/                # API route handlers
│   │   ├── __init__.py
│   │   ├── frictions.py
│   │   ├── initiatives.py
│   │   └── dashboard.py
│   └── services/               # Business logic
│       ├── __init__.py
│       ├── calculation.py
│       ├── classification.py
│       └── priority.py
├── requirements.txt
├── alembic.ini                 # Migrations config
├── alembic/                    # Migration scripts
└── .env                        # Local env vars (gitignored)
```

### Guidelines

- Type hints en todas las funciones (parámetros y retorno)
- Pydantic schemas para validación de input/output (no dicts crudos)
- Servicios contienen lógica de negocio, routers solo HTTP handling
- Dependency Injection via FastAPI Depends()
- Async para endpoints, sync para servicios de cálculo (no I/O)

## Frontend (Angular / TypeScript)

### Naming Conventions

| Element | Convention | Example |
|---------|------------|---------|
| Components | kebab-case files, PascalCase class | `friction-list.component.ts` / `FrictionListComponent` |
| Services | kebab-case files | `friction.service.ts` |
| Interfaces/Models | PascalCase | `Friction`, `Initiative` |
| Variables | camelCase | `frictionList` |
| Constants | UPPER_SNAKE_CASE | `API_BASE_URL` |

### Project Structure

```
frontend/src/app/
├── core/                # Servicios singleton, interceptors
│   └── services/
├── features/            # Feature modules
│   ├── frictions/
│   ├── initiatives/
│   └── dashboard/
├── shared/              # Componentes y pipes compartidos
│   ├── components/
│   └── models/
└── app.component.ts
```

### Guidelines

- Standalone components (Angular 18+)
- Signals para estado local en componentes
- Services para estado compartido
- Tipado estricto (strict: true en tsconfig)
- Evitar `any` — siempre tipar

## Error Handling

### Backend

- HTTPException con status codes apropiados
- Pydantic validation errors → 422 automático
- Custom exception handlers para errores de dominio

### Frontend

- HTTP interceptor para manejo centralizado de errores
- Toast notifications (PrimeNG) para errores de usuario
- Console.error para errores de desarrollo

## Import Order

### Python

```python
# 1. Standard library
# 2. Third-party (fastapi, sqlalchemy, pydantic)
# 3. Local imports (app.models, app.services)
```

### TypeScript

```typescript
// 1. Angular core
// 2. Angular modules
// 3. Third-party (PrimeNG, rxjs)
// 4. App core/shared
// 5. Relative imports
```
